# U6 ‚Äî Surfacing the full feature set

> Dependent stream (board sequencing step 6, [`README.md`](README.md) ¬ß"Recommended sequencing"; deps **U0**, **U3**; also **L0**, **U2**, **U4**). `latent`'s data model already carries more than the panel exposes: [`Adjustments`](../../latent-edit/src/lib.rs) has an `hsl: Option<Hsl>` and a `channel_mixer: Option<ChannelMixer>` the GUI **never** reads or writes ([`latent-edit/src/lib.rs:376-378`](../../latent-edit/src/lib.rs); absent from `gui.rs`); a `Mask` is a **list** of shapes combined by per-shape `ops` ([`latent-edit/src/lib.rs:60-95`](../../latent-edit/src/lib.rs)) but the UI only edits `shapes[0]` and never touches `ops` (`gui.rs:1134-1227`, `local_shape_block`); a `LocalAdjustment` holds a **full** `Adjustments` ([`:35-42`](../../latent-edit/src/lib.rs)) yet the panel offers only opacity + exposure + saturation (`gui.rs:1102-1127`); lens correction is **force-applied** silently on a fresh document (`gui.rs:47-53`, `auto_lens_profile` `:89-102`); and white balance is two raw `‚àí1..1` sliders with no Kelvin, eyedropper, or presets (`gui.rs:727-743`, `white_balance_block`). This epic is pure surfacing ‚Äî it **wires the existing model to the UI** and changes one default (lens off). It is the convergence point of the build-up epics: it **reuses U3's** `adjust_slider` + collapsible sections + begin/commit history, **U2's** canvas-handle framework (mask handles, pick-pixel), **L0's** working detector, and **U4's** config ‚Äî so it adds almost no new primitives, only new panels bound to model fields.
>
> **Shared surface:** the right-hand control panel ([`latent-app/src/gui.rs`](../../latent-app/src/gui.rs), `update` `:407-493`, which after U0-C1 becomes a `gui/panels/` module tree) and the `Adjustments`/`Mask`/`Geometry` data model ([`latent-edit/src/lib.rs`](../../latent-edit/src/lib.rs)). Take the cards in dependency order: **C1** (HSL) and **C2** (channel mixer) build the two missing global panels as reusable widgets; **C3** reuses every global control ‚Äî including C1/C2 ‚Äî against a *local's* `Adjustments` so a local gains the full surface; **C4** turns the single-shape mask editor into a shape **list** with ops, wiring U2's per-shape handles; **C5** removes the silent lens auto-apply and replaces it with an opt-in panel backed by L0's detector; **C6** upgrades white balance to Kelvin + eyedropper + presets over the same `WhiteBalance{temp,tint}` model. **C1/C2/C3 are the same widget reused**; **C4/C5/C6 each consume a U2 canvas affordance**.
>
> **Render math is untouched (Global rule 2).** Every card here is a *binding* ‚Äî it sets `Adjustments`/`Mask`/`Geometry` fields the engine already interprets. No card changes `latent_pipeline::render`, `to_color_image`, or any shader. The one model-level change is **additive** and **forward-compatible** (Global rule 4): U6-C5's lens-enabled gate is a new `#[serde(default)]` field so old sidecars still load. The Kelvin‚Üîoffset and eyedropper math in U6-C6 lives **entirely in the UI** ‚Äî it computes a `WhiteBalance{temp,tint}` offset and hands it to the unchanged engine (`apply_global` `:477-481`); it never reaches into the pipeline.
>
> **Authoritative grounding** (the model the panels bind to): `Hsl` ([`latent-edit/src/lib.rs:401-412`](../../latent-edit/src/lib.rs)) ‚Äî `bands: [[f32; 3]; 8]`, eight evenly-spaced hue bands **red, orange, yellow, green, aqua, blue, purple, magenta** (index 0‚Ä¶7), each `[hue, sat, lum]`; `hue` shifts the band's hue in turns, `sat`/`lum` scale by `1 + value`; all-zero is neutral; a pixel is driven by its two nearest bands. `ChannelMixer` ([`:425-439`](../../latent-edit/src/lib.rs)) ‚Äî `matrix: [[f32; 3]; 3]` (rows = output R/G/B, columns = input R/G/B), default identity, plus `preserve_luminosity: bool` (Photoshop's checkbox: when on, each row is normalized to sum 1 before applying so a neutral gray maps to itself). `WhiteBalance` ([`:443-448`](../../latent-edit/src/lib.rs)) ‚Äî `{ temp, tint }`, both `0` neutral, finite-only with **no bound** (sanitized `finite_or`, `:730-733`); the engine lowers it to per-channel gains `(1+temp, 1-tint, 1-temp)` with **green anchored** ([`latent-pipeline/src/lib.rs:477-481`](../../latent-pipeline/src/lib.rs)). `Mask`/`MaskOp` ([`:60-107`](../../latent-edit/src/lib.rs)) ‚Äî `shapes: Vec<MaskShape>`, parallel `ops: Vec<MaskOp::{Add,Subtract,Intersect}>` (`shapes[0]` is always the base, its op ignored; a missing op defaults to `Add`), `invert`. `LensProfile`/`Geometry.lens` ([`:609-671`, `:545`](../../latent-edit/src/lib.rs)) and the L0 detector ([`latent-lens/src/lib.rs:79-116`](../../latent-lens/src/lib.rs), `Metadata` `:166-209`).

- Heads-up:
  - **The engine's WB is a ‚àí1..1 offset, not absolute Kelvin ‚Äî the Kelvin/tint/eyedropper math is a UI-side mapping onto that offset.** `WhiteBalance { temp, tint }` ([`:443-448`](../../latent-edit/src/lib.rs)), both `0` neutral, finite-only with **no documented bound** (sanitized `finite_or`, `:730-733`). The engine lowers it to per-channel gains **`(rg, gg, bg) = (1.0 + temp, 1.0 - tint, 1.0 - temp)`** with **green as the anchor** ([`latent-pipeline/src/lib.rs:477-481`](../../latent-pipeline/src/lib.rs)). This single line is the ground truth for every WB tool here ‚Äî read it, don't re-derive: red and blue move **antisymmetrically** with `temp` (warmer = more red, less blue), green moves with `tint` only. **This card must NOT change that line** (Global rule 2) ‚Äî it computes a `{temp,tint}` and hands it to the unchanged engine.
  - **Kelvin‚Üîoffset mapping ‚Äî define it, document it, keep it reversible.** There is no absolute Kelvin in the model, so pick a **monotonic, invertible UI mapping** from a Kelvin slider (e.g. `2000..=12000 K`, neutral at a chosen reference `T0` ‚âà 5500 K "as shot/daylight") to `temp`, and from a tint slider (`‚àí150..=150` green‚Üîmagenta, neutral 0) to `tint`. The simplest faithful mapping that respects the engine's antisymmetric `(1+temp, 1, 1-temp)`: map Kelvin to `temp` by a smooth monotonic function anchored at `temp=0` for `T0` ‚Äî e.g. `temp = k_t * (1/T0 - 1/T)` (mired-difference, which is perceptually even and trivially invertible: `T = 1/(1/T0 - temp/k_t)`), with `k_t` chosen so the slider ends land in a sane `temp` range (roughly `¬±0.5`, well inside the unbounded field). Map tint linearly: `tint = k_u * (slider ‚àí 0)`. **Document the mapping and its inverse in a code comment** (the gotcha's explicit ask) so the displayed Kelvin‚Üîstored-offset relationship is checkable and so loading a sidecar's `temp` can show a Kelvin back. Keep the two **raw** `temp`/`tint` sliders available too (advanced), or at least ensure the Kelvin/tint sliders write the exact same `WhiteBalance{temp,tint}` field ‚Äî no second representation stored.
  - **Gray eyedropper ‚Äî neutralize a picked patch, math from the engine's gain formula.** When the user clicks a neutral target on the canvas (U2-C1 pick-pixel + U1-C5's RGB readout gives the **linear working-RGB** sample `[r,g,b]` at that pixel **as currently rendered**), compute the `{temp,tint}` that makes that patch neutral. Because the engine multiplies by `(1+temp, 1-tint, 1-temp)` and the *current* preview already has the current WB baked in, solve for the **delta** offset that equalizes the sample's channels: to make `r' = b'` and both equal `g'`, you need `r*(Œîrg) == g*(Œîgg) == b*(Œîbg)` under the engine's form. With green anchored, the neutralizing offset from a sample `[r,g,b]` is `temp_new` such that `r*(1+temp) = b*(1-temp)` ‚Üí `temp = (b ‚àí r)/(b + r)` (the value that balances red and blue), and `tint` such that green matches the red/blue average: `gg = 1 - tint` chosen so `g*(1-tint)` equals the balanced R/B level ‚Üí `tint = 1 ‚àí (balanced_rb / g)` (guard `g>0`). **Compose with the current WB** (the sample already includes it) so the result is the *absolute* `{temp,tint}` that neutralizes that patch ‚Äî i.e. add the delta to the current offset, or sample from the WB-off preview (U1-C4's settings-off preview already exists) to get the pre-WB pixel and solve directly. Document which space you sample (pre- or post-WB) ‚Äî it determines whether you add or replace. Clamp/guard for `g‚âà0` and non-finite (mirror the engine's `finite_or`, `:731-732`). The result is **reversible** (it's just a `{temp,tint}`); a manual check confirms the picked patch goes neutral.
