# E6 — Resampling, geometry GPU parity & convergence

> The **convergence epic.** Realizes the resampling/minification + GPU-parity decisions of [`../decisions/05-geometry-and-optics.md`](../decisions/05-geometry-and-optics.md) (**§2.7** higher-order interp + prefilter, **§2.9** GPU `w≤0` guard + `warp.wgsl`, **N1** auto-scale), the output-sharpening pass of [`../decisions/04-spatial-and-frequency.md`](../decisions/04-spatial-and-frequency.md) (**§2.6**), and the CPU/GPU-backend findings of [`../decisions/code-review/03-cpu-and-gpu-backends.md`](../decisions/code-review/03-cpu-and-gpu-backends.md) (**F1** guard, **F3**/**TG** test gap, **F4/F5** pooling + recoverable readback, **F6** port `combine`/`apply_radial_gain`) under the **CR-02 M4** Test/CPU/GPU contract.
>
> **Depends on:** **E3** (the `Warp`/`lens_radial` CPU math — `Warp::map`/`map_channel` is the single reference `warp.wgsl` mirrors; do not fork it) and **E5** (the L\*-perceptual luma-sharpen form — E6-C3 reuses it for the output pass). Do not start C2/C3 before those land or the GPU shader/output-sharpen will mirror stale math.
>
> **This epic owns interpolation + minification prefilter for the whole codebase (Global rule 5).** E6-C1 is the *single* owner of the higher-order kernel + prefilter; every other minifying caller (the geometry warp/resample, N1 auto-scale, any future downscale) routes through it rather than re-implementing. C2 makes the GPU match C1 bit-for-bit; the equivalence harness (C6) is the guard rail that keeps them from ever drifting.
>
> **Shared surface:** `latent-cpu/src/lib.rs` (`sample_bilinear:273`, `resample:101`, `warp:118`, `apply_radial_gain:145`, `combine:76`); `latent-gpu/src/{lib.rs,resample.wgsl}` (+ new `warp.wgsl`, `combine.wgsl`); `latent-pipeline/src/lib.rs` (`apply_geometry:689`, `render:353`, `Warp`/`Transform`/`RadialGain`). Take the cards roughly in order: **C1** lands the CPU interpolator+prefilter (the owner), **C2** mirrors it on GPU + adds the guard + `warp.wgsl`, **C3** adds the post-geometry output sharpen, **C4** adds auto-scale (through C1's prefilter), **C5** ports `combine`/`apply_radial_gain`, **C6** builds the three-backend equivalence harness that locks the lot in, **C7** does the GPU robustness/perf (recoverable readback + resident buffer + pooling). **C5/C6/C7** can interleave once C1/C2 are green.

### E6-C1 — Higher-order interpolation (Lanczos/bicubic) + minification prefilter (CPU)
- Implements: IP-05 §2.7            Priority: High
- Crates/files: `latent-cpu/src/lib.rs` (replace `sample_bilinear:273`; its two callers `resample:112` and `warp:134/138`); the doc-comment block `:264-272` (rewrite — it currently promises "bilinear, no prefilter, unit-scale only").
- Depends on: — (E3 only for the `Warp` it feeds — the interpolator itself is independent)             Blocks: E6-C2, E6-C3, E6-C4, E6-C6
- Heads-up:
  - **This card is the single owner of interpolation + minification prefilter (Global rule 5). Every other minifying caller — the GPU mirror (C2), N1 auto-scale (C4), the output-sharpen aliasing it compensates for (C3) — depends on this one and must not re-implement the kernel or the prefilter. Get the contract (kernel, support, prefilter sizing, border, color space) nailed down here; the rest of the epic mirrors it.**
  - **Two distinct problems, both must be solved (register §2.7):** (a) **magnification / general resampling** quality — replace the 2-tap **bilinear** with a **higher-order kernel**: Lanczos-3 (windowed sinc, 6×6 support, the RawTherapee/darktable default) or a Catmull-Rom / Mitchell bicubic (4×4 support). Pick one and document it; Lanczos-3 is the register's lead suggestion for sharpest magnification. (b) **minification aliasing** — higher-order interpolation *does not* fix it (Szeliski §3.5.2): when the local map shrinks the source, a point sample (of any kernel) undersamples and aliases. Add a **prefilter** sized to the local minification.
  - **Prefilter — pick one and size it from the local map Jacobian.** The map is `Warp::map`/`Transform::map` (output→source); its 2×2 Jacobian `J` at each output pixel tells you the source footprint of that output texel. Options, easiest→best: **(i) Jacobian-sized box / area average** — average the source over the parallelogram `J` maps the unit output texel to (degenerates to a box of side ≈ `max(σ_x, σ_y)` where `σ = ‖J column‖`); cheap, isotropic-ish. **(ii) MIP / trilinear** — build a mip pyramid of the source once, pick the level from `log2(max singular value of J)`, trilinear-blend between levels, then interpolate within the level with the C1 kernel. **(iii) EWA (elliptical weighted average)** — the gold standard: an anisotropic Gaussian footprint from `J` (Szeliski §3.5.2 / Heckbert). For a first cut a **Jacobian box or MIP is acceptable**; leave EWA as a documented upgrade. The footprint must be computed from the *actual* per-pixel Jacobian (distortion/keystone minify *locally*, non-uniformly — a global scale is wrong at the corners).
  - **Do not regress the magnifying/unit-scale path.** When `J` is ≥ unit scale (magnification or identity), the prefilter must collapse to *no* averaging (mip level 0 / box side 1) so you get pure higher-order interpolation, not a softened result. The prefilter only engages where the map locally minifies. Identity/crop/rotation (the cases the old comment said bilinear was "only" for) must come out as sharp as or sharper than before.
  - **Preserve the invariants the register marks correct:** **single resample** (the composed homography∘radial∘CA lookup stays one interpolation — do not add a second warp pass; the prefilter is a *sampling-time* footprint, not a separate resize), **zero-border** (a tap outside the source contributes black → past-border fades to black; keep this exactly — both the kernel taps and the prefilter footprint clamp/zero at the edge identically to today's `at` closure at `:279-285`), and **linear-light sampling** (the working image is already linear; do not gamma-correct — unchanged). Single-resample + zero-border are *verified correct*; only the kernel and the prefilter are new.
  - **CA path:** `warp` samples each channel at its own radius when `has_chromatic()` (`cpu:131-135`). The new sampler must work per-channel too — either a per-channel scalar-returning sampler, or sample all three and index. Keep the "one fetch per channel only when chromatic" structure (no cost when CA is off).
  - **GPU lockstep (Global rule 1):** C2 mirrors this on the GPU and **must match bit-close** (the C6 harness enforces a tolerance). So choose a kernel + prefilter that is *portable to WGSL* — avoid CPU-only tricks (e.g. arbitrary recursion, large dynamic tables) that you can't reproduce in a shader. Document the exact kernel coefficients / Lanczos `a` / cubic B,C and the prefilter footprint formula so C2 is a transcription, not a reinvention.
- Acceptance:
  - `sample_bilinear` is replaced (or superseded) by a higher-order sampler; `resample` and `warp` use it. A **test** `lanczos_sharper_than_bilinear_on_magnify` (or `bicubic_…`) upsamples a known edge and asserts the new kernel's edge response is sharper than the old bilinear (e.g. higher gradient at the edge / closer to the analytic ramp) — fails with bilinear, passes after.
  - A **test** `minify_prefilter_suppresses_aliasing` resamples a high-frequency pattern (fine checker / zone plate) through a strongly minifying map and asserts the output aliasing energy (variance of a region that should be near-flat post-filter, or a moiré metric) is far below the no-prefilter result — the prefilter's reason to exist.
  - A **test** `unit_scale_resample_not_softened` (identity / pure rotation / crop) asserts the prefilter is inert at ≥ unit scale: the result is at least as sharp as the pre-card bilinear on those transforms (no accidental blur on the non-minifying path).
  - Zero-border preserved: a **test** that an output pixel mapping outside the source still reads black (regression of the existing fade-to-black). Single-resample preserved (no second pass introduced). The kernel coefficients and prefilter footprint formula are documented in-code for C2 to mirror. `cargo test --workspace`, `cargo clippy --all-targets` (0 warnings), `cargo fmt --check` green.

### E6-C2 — GPU resample `w≤0` guard + `warp.wgsl` + matching interpolation/prefilter
- Implements: IP-05 §2.9, CR-03 **F1** (+ **F2**/**F9** doc nits folded in)            Priority: High
- Crates/files: `latent-gpu/src/resample.wgsl` (guard + C1 kernel/prefilter); new `latent-gpu/src/warp.wgsl` (+ pipeline/bind-group wiring in `latent-gpu/src/lib.rs`); `latent-gpu/src/lib.rs` (`warp:539` stops delegating to CPU; new `WarpParams`/std140 struct; `resample:508`); the `ResampleParams`/`MapParams` std140 comments.
- Depends on: E6-C1 (the kernel + prefilter to mirror), E3 (the `Warp::map`/`map_channel` math `warp.wgsl` transcribes)             Blocks: E6-C6
- Heads-up:
  - **Three things in one card, all kept bit-aligned with C1/E3 (Global rule 1): (1) the `w≤0` guard — the actual CR-03 F1 / IP-05 2.9 divergence; (2) `warp.wgsl` — the missing GPU radial/CA path; (3) the C1 higher-order kernel + minification prefilter, mirrored. The C6 harness is the guard rail; keep it green at every step.**
  - **(1) The guard (the headline bug).** `resample.wgsl:46-48` computes `w`, `sx`, `sy` *unconditionally* with no guard, while the CPU reference `Transform::map` (`pipeline:137-142`) returns `(-1,-1)` when `w ≤ 0` and the CPU `resample` relies on it. At extreme keystone corners the GPU samples real pixels or, at `w == 0`, feeds `inf`/`NaN` into `floor`/`i32` (undefined in WGSL → garbage). It **is reachable end-to-end** (keystone/straighten with *no lens block* routes `apply_geometry:748` → `backend.resample` → this shader). Add, *before* the divide at `:46`:
    ```wgsl
    if w <= 0.0 {
        let o = (gid.y * p.out_width + gid.x) * 3u;
        dst[o] = 0.0; dst[o + 1u] = 0.0; dst[o + 2u] = 0.0;
        return;
    }
    ```
    The same guard goes in `warp.wgsl`'s map (mirroring `Warp::map:252`). **F2 (verified correct):** keep `floor`-before-`i32` (`i32(floor(sx))` matches CPU `floor as i32`); do *not* "simplify" to `i32(sx)` — add the one-line comment so a future editor doesn't. **F9 (doc nit):** add the std140 comment on `ResampleParams`/the WGSL `Params` pad (`resample.wgsl:22-24`, Rust `:69`) and the new `WarpParams` — "rounds to a 16-byte multiple to satisfy std140 struct alignment; do not remove".
  - **(2) `warp.wgsl` (the missing path).** Today `GpuBackend::warp:539` just delegates to `self.cpu.warp` — lens distortion/CA never touches the GPU. Write `warp.wgsl` mirroring `Warp::map`/`map_channel` (`pipeline:250-282`): homography + `w≤0` guard → if `radial != 0`, the **radial Horner** `s = 1 + r(d0 + r(d1 + r(d2 + r·d3)))` about `center` with `inv_norm` (and, after E3-C3, the **Newton inversion** for POLY3/POLY5 — transcribe E3's CPU loop; PTLENS stays the direct multiply) → per-channel CA scale of the offset from `center` (`map_channel`), dispatching the channel separately only when `has_chromatic()`. Add a `WarpParams` uniform (`m[9]`, `center[2]`, `inv_norm`, `radial[4]`, `channel_scale[3]`, + std140 pad; mind the `array<f32,N>`/`vec3` std140 stride trap — keep scalars 4-aligned and contiguous like `ResampleParams` already does). Wire its pipeline/bind group alongside `resample_pipeline`. **`Warp::map`/`map_channel` (CPU, E3) is the single reference — transcribe, do not re-derive.**
  - **(3) Mirror C1's kernel + prefilter** in *both* `resample.wgsl` and `warp.wgsl`: the same higher-order kernel (Lanczos-`a` / cubic B,C — use the exact coefficients C1 documented), the same Jacobian-box/MIP prefilter, the same zero-border `fetch` (`resample.wgsl:31-37` already zeros out-of-bounds — keep it identical for every tap). For the prefilter, the per-pixel Jacobian can be computed analytically from the map (differentiate the homography/radial) or by finite-differencing `map` at `±0.5` in `gid` — pick what matches C1. If C1 uses MIP, build the pyramid on the GPU or upload mip levels; if a Jacobian box, sum the footprint in-shader. **The CPU and GPU must agree within the C6 tolerance** — same kernel, same footprint, same edge handling.
  - **Performance note (couples to C7):** `warp.wgsl` replaces a CPU round-trip in the geometry stage; once it lands, `apply_geometry`'s warp runs GPU-resident (C7 keeps the buffer resident around it). Do not regress to a CPU bounce.
- Acceptance:
  - **Primitive parity** `gpu_resample_w_le_0_matches_cpu`: a transform with behind-plane corners (e.g. `keystone_transform(_, 0.8, 0.8)`) — those output pixels read **black** on GPU exactly as CPU (no NaN/garbage); fails before the guard, passes after.
  - **`warp.wgsl` parity** `gpu_warp_matches_cpu_radial` and `gpu_warp_matches_cpu_ca`: a radial-distortion warp and a CA warp render on the GPU within the equivalence tolerance of the CPU `Warp` reference (the path that was CPU-only). Both `resample.wgsl` and `warp.wgsl` use the C1 kernel + prefilter.
  - **Kernel/prefilter parity** `gpu_minify_prefilter_matches_cpu`: the minifying case from C1 matches CPU within tolerance (the two backends share one interpolation+prefilter contract). A **border** parity test (`gpu_resample_border_fade_matches_cpu`) where output maps on/over the source edge — written against the *final* (Lanczos/prefilter) interpolator, per CR-03 TG(2).
  - F2 comment present; std140 comments on `ResampleParams`/`WarpParams` (F9). `cargo test --workspace` (GPU tests skip cleanly with no device), clippy 0-warnings, fmt clean.

### E6-C3 — Output-sharpening pass (post-geometry, perceptual luma)
- Implements: IP-04 §2.6            Priority: Medium

- Heads-up:

The whole point: create the sharpening overshoots at *output* resolution so the resampler doesn't alias them away. It reuses E5-C3's exact L\*-perceptual luma-sharpen form (luma-only, recombine around the sharpened L\* — no edge fringing, symmetric overshoot); do not invent a second sharpen math.**

  - **Settings/forward-compat (E7 rule):** add the output-sharpen control with `#[serde(default)]` so old sidecars load and an absent/zero setting is a no-op (the default render must stay identical to pre-card for default settings — assert this).

### E6-C4 — Auto-scale-to-fill option
- Implements: IP-05 **N1**            Priority: Medium
- Crates/files: `latent-pipeline/src/lib.rs` (`apply_geometry:689` — compute a fill scale and fold it into the homography/`Warp.m` *before* the single resample; `lens_radial`); `latent-edit/src/lib.rs` (an auto-scale toggle in `Geometry`, `#[serde(default)]`).
- Depends on: E6-C1 (auto-scale may **minify** → must go through C1's prefilter), E3 (the distortion map it scales to fill)             Blocks: —
- Heads-up:
  - **A feature: a lensfun `GetAutoScale`-equivalent so distortion/keystone/straighten don't leave black border wedges. The critical constraint (register N1): the fill scale can *minify* the image (a barrel-distortion correction that pushes corners out needs a scale-down to fill), so the resample that applies it MUST go through C1's minification prefilter — this is exactly why N1 depends on C1.**
  - **Compute the fill scale like lensfun's `GetAutoScale`.** Trace the geometry map at a set of boundary samples (the four corners and points along each edge — corners alone miss the worst case for strong distortion) from output→source; find the scale `s` about the image center such that the scaled output frame is fully covered by valid source (no sample lands outside). lensfun samples the frame edges and takes the max required scale; mirror that. Apply `s` as a center-preserving scale folded into the **same** homography (`Warp.m` / `Transform.m`) so it stays **one resample** (Global rule 5 — do *not* add a second resize pass). The output extent is unchanged (the source frame size, per `apply_geometry`'s current contract); auto-scale changes *what fraction of the source fills it*, removing the black wedges.
  - **It composes with everything in `apply_geometry`.** Straighten's bbox expansion, keystone, and lens distortion all already compose into one homography/`Warp` (`:715-746`); fold the auto-scale into that same matrix *before* the resample/warp call — after the homography is built, before `backend.warp`/`backend.resample`. Order: distortion/keystone/straighten define the map → auto-scale finds the fill scale of *that* map → scale folds in → single resample (now possibly minifying → C1 prefilter engages automatically). Interacts with crop (`:751`): auto-scale runs before crop, as a frame-fill, not a crop.
  - **Off by default; opt-in toggle.** Add to `Geometry` with `#[serde(default)]` (E7 forward-compat); default off so existing renders are unchanged (the current doc at `apply_geometry:686` says "there is no auto-scale-to-fill" — update it). When off, behavior is exactly today's (black wedges the user crops away).
  - **GPU lockstep:** auto-scale is pure homography math folded into `Warp.m`/`Transform.m` — no new primitive, so the GPU path inherits it for free *provided* the scaled (minifying) resample uses C2's mirrored prefilter. Add it to a C6 end-to-end case so CPU/GPU agree on the auto-scaled render.
- Acceptance:
  - `auto_scale_fills_distortion_border`: a barrel/keystone correction that *without* auto-scale leaves black corner wedges, *with* auto-scale on has no black border (every output pixel maps inside the source) — at the cost of a slight crop/minify. Fails (wedges present) when the toggle is off.
  - `auto_scale_goes_through_prefilter`: when the fill scale minifies, the result uses C1's prefilter (e.g. no aliasing on a high-frequency source under auto-scale — contrast with a hypothetical point-sampled scale). Proves the C1 dependency is real, not nominal.
  - `auto_scale_off_is_unchanged` / serde-default: with the toggle absent/off the render is identical to pre-card and an old sidecar loads. Stays a **single** resample (no second resize pass introduced). `cargo test --workspace`, clippy 0-warnings, fmt clean.

### E6-C6 — CPU/GPU + Test/CPU/GPU equivalence harness; close the test gap
- Implements: CR-03 **F3** + **TG(3)(4)**, CR-02 **M4**            Priority: High
- Crates/files: `latent-gpu/src/lib.rs` (tests, near `render_matches_cpu_across_the_pipeline:748` and the standalone resample tests `:688-741`); a shared conformance helper (tolerance + backend-pair driver) usable by Test/CPU/GPU; fixtures shared with E3-C6 where applicable.
- Depends on: E6-C1, E6-C2 (the guard + `warp.wgsl` + kernel the harness exercises)             Blocks: —
- Heads-up:
  - **This is the test half of the convergence epic and the reason F1 slipped through.
